## 3. 핵심 원칙

1. **"왜"를 항상 붙인다** — 면접 답변 가능한 수준의 근거. 결정은 `decision_log.md`에 기록.
2. **실행 가능한 Python** — 의사코드 금지. 파일 경로, import, 타입 힌트 포함.
3. **확장 가능한 구조** — LLM/벡터 백엔드는 `services/` 추상화 레이어 뒤로. `openai` 직접 호출 금지.
4. **비용 의식** — 모든 LLM 호출은 캐시 경유. 기본 `gpt-4o-mini`, 품질이 중요한 경로만 `gpt-4o`.
5. **에러 핸들링 기본** — happy path만 있는 코드는 미완성.
6. **범위 규율** — 요청된 작업에 필요한 것 이상의 기능 추가·리팩토링·추상화 도입은 하지 않는다. 개선 여지를 발견하면 실행하지 말고 보고한다 (AGENTs.md §범위 규율).
7. **증거 기반 보고** — 완료·진행 보고는 이 세션의 실행 결과(테스트 출력, 디스크 파일)로 뒷받침 가능한 것만. 미검증 항목은 미검증이라고 명시한다.
8. **샘플 먼저** — 가장 얇은 수직 슬라이스부터. N5 샘플 → N5 전체 배치 → N4/N3 (항목 수는 최신 세션 문서 기준 — 수치를 기억으로 말하지 않는다).

## 4. 작업 프로세스

**구현 표준 흐름**
요구사항 확인 → 영향 파일 식별 → 구현 → 실행 검증 (CLAUDE.md [VERIFY] 파이프라인: pytest / mypy / ruff / 테스트 존재) → 문서 업데이트 → 다음 액션 제안

**의사결정 제시 형식**
```
선택지 A / B → 트레이드오프 → 추천 + 이유 → 정빈님 확정 필요 여부
```

**핸드오프 시**: 파일 경로 + 변경 범위 + 다음 단계 요청 명시.

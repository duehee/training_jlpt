## 1. 신규 사용자 온보딩

핵심 원칙: **로그인 장벽을 진단 뒤로 미룬다.**

```text
웹 앱 진입
  ↓
익명 진단 시작 (로그인 불필요)
  ↓
진단 테스트 (10문항 / N5×4 · N4×3 · N3×3 / 4지선다)
  ↓
답안 순차 제출
  ↓
진단 완료 → 레벨 판정 + 약점 문법 요약
  ↓
결과 확인
  ↓
로그인/가입 유도
  ↓
가입 완료 → 진단 결과 자동 승계 → 학습 시작
```

**참조 다이어그램**: `docs/new_user_onboarding_flow.svg`

**관련 문서**:
- 진단 API 상세 → `api_endpoints.md` 3장
- 익명/회원 데이터 구조 → `database_schema.md` 3장, 4~7장
- 익명 → 로그인 데이터 승계 구현 → `database_schema.md` 4장

## 2. 핵심 학습 루프

로그인 사용자가 학습 세션을 진행하는 메인 흐름입니다.
**LangGraph 에이전트 워크플로우**로 구현됩니다.

```text
학습 포인트 선정
  - 기본: 약점 기반 자동 추천
  - 예외: 사용자 수동 변경
  ↓
문법 청크 Retrieval (grammar + 관련 compare)
  ↓
설명 + 예문 생성 (LLM)
  ↓
이해도 확인 문제 출제 (4지선다 1문항)
  ↓
답안 제출 및 판정
  ↓
분기
  ├─ 정답: 다음 학습 포인트로 이동
  └─ 오답: 다른 각도로 재설명 → 재출제
         └─ 연속 3회 오답 시 세션 종료, 다른 포인트 제안
```

**참조 다이어그램**: `docs/core_learning_agent_flow.svg`

**관련 문서**:
- 학습 API 상세 → `api_endpoints.md` 6장
- 청크 JSON 구조 → `database_schema.md` 15장
- LLM 캐싱 전략 → `database_schema.md` 14장

## 3. 재방문 사용자 플로우

핵심 원칙: **"뭐부터 할까" 고민 없이 즉시 이어하기.**

```text
서비스 재진입 (로그인 상태)
  ↓
last_session 조회
  ├─ 있음: "최근에 'は vs が'를 학습하셨어요. 이어서 할까요?"
  └─ 없음: 약점 기반 첫 학습 포인트 추천
  ↓
사용자 선택
  ├─ 이어하기: 해당 세션 재개
  └─ 새로 시작: 추천 포인트 또는 수동 선택
  ↓
핵심 학습 루프 진입 (2장)
```

**관련 문서**:
- 이어하기 API → `api_endpoints.md` 7장
- `last_session` 테이블 → `database_schema.md` 12장

## 4. Phase별 범위

### Phase 1 (1~2개월차) — 현재 단계
**목표**: 진단 + RAG 기반 문법 설명 에이전트의 독립 데모 완성

**포함 기능**:
- 익명 진단 테스트 (10문항, N5~N3)
- 레벨 판정 + 약점 분석
- 로그인/가입 및 데이터 승계
- 약점 기반 학습 포인트 추천
- Retrieval 기반 문법 설명 (grammar + compare 청크)
- 이해도 확인 문제 + 정오답 분기
- last_session 기반 이어하기

**데이터 범위**:
- N5 문법 청크 84개
- N5 비교 청크 21쌍
- N4/N3는 진단 문제 범위에만 사용

**UI**: Streamlit 데모 (프론트 공수 최소화)

### Phase 2 (3개월차)
**목표**: TTS 기반 청해 훈련 추가
- 청해 청크 (listening) 추가
- TTS API 연동
- 청해 문제 5개 유형 생성
- N4/N3 문법 데이터 확장

### Phase 3 (4~5개월차)
**목표**: 고도화 + 상품화 검토
- 오답 패턴 분석 파이프라인
- 학습 평가 지표 (eval 프레임워크)
- 통합 대시보드
- 독해 청크 (reading) 추가
- BM 모델 검토
- 배포 방식 결정 (PWA vs 네이티브)

### Phase 범위 조정 원칙
- Phase 1 완료가 최우선. Phase 2/3가 지연되면 축소 가능.
- 새 기능 제안은 `product_overview.md` 6장 "기능 채택 판단 기준" 통과해야 함.

## 5. 데이터 단위 정의

학습 콘텐츠는 **청크(chunk) 단위**로 관리됩니다.

| 청크 타입 | 설명 | chunk_type 값 | Phase |
|-----------|------|--------------|-------|
| 문법 청크 | 단일 문법 포인트 설명 | `grammar` | Phase 1 |
| 비교 청크 | 혼동 문법 쌍 비교 설명 | `compare` | Phase 1 |
| 청해 청크 | TTS 스크립트 + 청해 문항 | `listening` | Phase 2 |
| 독해 청크 | 독해 지문 + 해설 | `reading` | Phase 3 |

**청크 상세 스키마**: `database_schema.md` 12장, 14장 참조.

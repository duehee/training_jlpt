# 서비스 플로우

> 최종 업데이트: 2026-04-14
> 이 문서는 사용자 여정과 Phase별 범위를 정의합니다.

---

## 1. 신규 사용자 온보딩

```text
웹 앱 진입
  ↓
익명 세션 생성
  ↓
진단 테스트 시작
  - 총 10문항
  - N5: 4 / N4: 3 / N3: 3
  - 문법 중심 객관식
  ↓
진단 완료
  ↓
레벨 판정 및 약점 문법 표시
  ↓
로그인 또는 회원가입 유도
  ↓
학습 시작
```

참고 다이어그램: `docs/newUserOnboardingFlow.svg`

---

## 2. 핵심 학습 루프

```text
다음 문법 포인트 선택
  - 에이전트가 약점 기반 추천
  - 사용자가 변경 가능
  ↓
관련 문법 청크 retrieval
  - 레벨 필터
  - 문법 포인트 필터
  ↓
설명과 예문 생성
  ↓
확인 문제 제시
  ↓
분기
  - 정답: 다음 포인트로 이동
  - 오답: 다른 각도로 재설명 후 재시도
```

참고 다이어그램: `docs/coreLearningAgentFlow.svg`

---

## 3. 재방문 사용자 플로우

```text
서비스 재진입
  ↓
마지막 학습 지점 불러오기
  ↓
이어하기 안내
  ↓
학습 재개
```

이어하기 경험은 DB의 `last_session` 상태를 기준으로 합니다.

---

## 4. Phase 범위

### Phase 1

- 진단 테스트와 레벨 판정
- 약점 문법 요약
- 문법 포인트 추천
- Retrieval 기반 문법 설명
- 확인 문제와 재시도 분기
- 마지막 학습 지점 이어가기

### Phase 2

- TTS 기반 청해 훈련
- 청해 스크립트와 문제 생성
- N4/N3 데이터 확장

### Phase 3

- 오답 패턴 분석
- 평가 파이프라인
- 학습 대시보드
- 상품화 검토

## 5. 데이터 단위

| 단위 | 설명 | Phase |
|------|------|-------|
| Grammar chunk | 단일 문법 포인트 설명 | Phase 1 |
| Comparison chunk | 헷갈리는 문법 쌍 비교 설명 | Phase 1 |
| Listening chunk | TTS 스크립트와 청해 자료 | Phase 2 |
| Reading chunk | 독해 지문과 해설 | Phase 3 |
